# project 2.0 Debug Tasks

## 说明

这份清单用于跟踪 `project 2.0` 当前阶段的运动控制与定位模块调试工作。

状态约定：
- `未开始`：任务已确认，尚未进入排查
- `排查中`：正在验证现象、读取数据、定位原因
- `已定位`：问题根因已确认，尚未完成修复或调参
- `已修复`：代码或参数已修改，待实车验证
- `待验证`：代码已修改，等待架空或落地测试
- `已完成`：功能、验证、记录均已闭环

默认执行顺序：
`1 -> 2 -> 3 -> 5 -> 6 -> 7 -> 8 -> 9 -> 10`

任务 4 为结构检查项，可穿插在 `3~7` 之间进行。

### Task 1 更新
- 日期：2026-03-21
- 当前现象：单轮正PWM测试时，右前和右后符合预期，但左前、左后在 `+PWM` 下仍然“向后转”。
- 怀疑原因：左侧两个轮子的 `MOTOR_FORWARD_LEVEL` 与当前机械安装方向相反。
- 已做排查：结合单轮测试模式和你的实测现象，确认问题只出现在左侧两个轮子。
- 结论：`MOTOR1_FORWARD_LEVEL` 和 `MOTOR3_FORWARD_LEVEL` 需要再次翻转；右侧保持不变。后续实测已经确认四个电机均正常。
- 下一步：进入编码器方向一致性验证。

### Task 2 更新
- 日期：2026-03-21
- 当前现象：左前和左后在电机正方向修正后，编码器符号也必须同步重配。
- 怀疑原因：编码器符号修正和电机方向修正是耦合的，不能只改其一。
- 已做排查：根据四轮单测结果，完成了 `ENCODERx_FORWARD_SIGN` 的联动修正。
- 结论：后续实测已经确认四个轮子的“正PWM -> 正编码器反馈”全部一致。
- 下一步：进入 IMU 通信与基础 yaw 验证。

### Task 3 更新
- 日期：2026-03-21
- 当前现象：原有 IMU 代码虽然存在 `imu_deal_data()` 链路，但主程序处于电机测试模式，没有真正初始化 IMU，也没有在固定周期内稳定更新 yaw；同时旧积分逻辑在静止时会直接把累计角度清零，无法用于保向验证。
- 怀疑原因：任务 3 之前更多是在为后续姿态控制预留接口，没有形成一个可独立验证“通信 + 零偏 + yaw”的测试闭环。
- 已做排查：检查了 `gyroscope.c/.h`、`main.c`、`imu963ra` 驱动和当前角度控制调用关系；随后切换到“IMU 基础姿态测试模式”，并完成了一轮静止、平移、原地旋转实测。
- 结论：
  - IMU 通信已经打通：静止时 `a=1`，说明初始化与零偏标定成功
  - 静止稳定性良好：静止放置时 `c=0`、`d=0`，没有明显自发漂移
  - 旋转跟踪正常：原地旋转时 `c` 和 `d` 都连续变化，说明 yaw 链路有效
  - 仍需进一步确认平移保向：来回平移时 `d` 出现数度变化，当前更像“手动平移时夹带了实际转动”，还不能直接判定算法失败
- 下一步：
  - 先做一次更严格的平移测试，尽量保证车头不扭转
  - 再做一次“清零 yaw 后完整旋转 360°”的终点误差测试
  - 如果 360° 终点误差明显偏大，再进入比例修正或更高频积分阶段

### Task 3 最新补充
- 日期：2026-03-21
- 当前动作：为了让第 3 项验证更可重复，`main.c` 已加入调试按键逻辑。
- 新增调试入口：
  - `KEY1` 短按：清零当前 `Yaw`
  - `KEY2` 短按：在车模静止时重新执行一次 IMU 初始化与零偏标定
- 当前蓝牙输出含义：
  - `a`：状态码，`1` 为正常，`11` 为刚刚清零 `Yaw`，`21` 为刚刚重标定，`0` 为 IMU 初始化失败
  - `b`：原始 `gyro_z`
  - `c`：去零偏后的角速度，单位 `0.01 deg/s`
  - `d`：积分得到的 `Yaw`，单位 `0.01 deg`
- 目的：避免每次做 360 度测试都必须重新上电，也避免旋转前残留角度影响精度判断。

## 当前关键原则

- 编码器方向未确认前，不进行 PID 参数整定。
