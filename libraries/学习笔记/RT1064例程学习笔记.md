## 第一阶段：基础 - GPIO 与最小闭环

### E01_gpio_demo：GPIO输入/输出/上拉

**核心内容**：学习GPIO的基本操作，建立"输入-处理-输出"的最小闭环。

#### 关键API

```c
// 初始化GPIO
// 参数：引脚, 输入/输出, 默认电平, 推挽/开漏
gpio_init(B9, GPO, 0, GPO_PUSH_PULL);        // 输出引脚
gpio_init(C4, GPI, 0, GPI_PULL_UP);            // 输入引脚(上拉)

// 设置电平
gpio_set_level(B9, 1);    // 设置高电平
gpio_set_level(B9, 0);    // 设置低电平
gpio_toggle_level(B9);    // 翻转电平

// 获取电平
gpio_get_level(C4);       // 返回 0 或 1

// 延时函数
system_delay_ms(200);     // 毫秒延时
system_delay_us(100);     // 微秒延时
```

#### 引脚表示方法
- `B9` = GPIO_B_9
- `C4` = GPIO_C_4
- 使用枚举类型 `gpio_pin_enum`

#### 模式选择
| 模式 | 说明 |
|------|------|
| `GPO` | 通用输出 |
| `GPI` | 通用输入 |
| `GPO_PUSH_PULL` | 推挽输出（能输出高低电平） |
| `GPO_OPEN_DRAIN` | 开漏输出（需外接上拉电阻） |
| `GPI_PULL_UP` | 输入带上拉 |
| `GPI_PULL_DOWN` | 输入带下拉 |

#### 实验现象
- 核心板蓝色LED按设定频率闪烁
- 可通过调试查看`gpio_status`变量读取按键状态

## 第二阶段：可观测性 - 串口与调试

### E02_uart_demo：串口通信

**核心内容**：学习UART通信，实现单片机与电脑的数据交互。

#### 硬件连接
```
模块引脚         单片机引脚
USB-TTL-RX       查看 DEBUG_UART_TX_PIN (默认 B12)
USB-TTL-TX       查看 DEBUG_UART_RX_PIN (默认 B13)
USB-TTL-GND      GND
USB-TTL-3V3      3V3
```

#### 关键API

```c
// FIFO队列初始化
fifo_init(&uart_data_fifo, FIFO_DATA_8BIT, uart_get_data, 64);

// UART初始化
// 参数：UART编号, 波特率, TX引脚, RX引脚
uart_init(UART_INDEX, UART_BAUDRATE, UART_TX_PIN, UART_RX_PIN);

// 开启接收中断
uart_rx_interrupt(UART_INDEX, ZF_ENABLE);

// 设置中断优先级
interrupt_set_priority(UART_PRIORITY, 0);

// 发送数据
uart_write_string(UART_INDEX, "Hello");      // 发送字符串
uart_write_byte(UART_INDEX, 'A');            // 发送单字节
uart_write_buffer(UART_INDEX, data, len);    // 发送缓冲区

// 接收数据（查询方式）
uart_query_byte(UART_INDEX, &get_data);      // 返回TRUE表示收到数据

// FIFO操作
fifo_write_buffer(&fifo, data, len);         // 写入FIFO
fifo_read_buffer(&fifo, data, &count, mode); // 读取FIFO
fifo_used(&fifo);                            // 获取已有数据量
```

#### 中断服务函数模板
```c
void uart_rx_interrupt_handler(void)
{
    uart_query_byte(UART_INDEX, &get_data);           // 查询接收
    fifo_write_buffer(&uart_data_fifo, &get_data, 1); // 写入FIFO
}
```

### E10_printf_debug_log_demo：日志与调试

**核心内容**：使用printf和zf_log进行调试输出，学习断言机制。

## 第三阶段：中断触发

### E06_exti_demo：外部中断

**核心内容**：学习外部中断、触发沿选择、中断优先级、去抖处理。

#### 触发模式
```c
EXTI_TRIGGER_RISING      // 上升沿触发
EXTI_TRIGGER_FALLING     // 下降沿触发
EXTI_TRIGGER_BOTH        // 双边沿触发
```

#### 关键API

```c
// 外部中断初始化
// 参数：引脚, 触发模式
exti_init(KEY1, EXTI_TRIGGER_RISING);
exti_init(KEY2, EXTI_TRIGGER_FALLING);
exti_init(KEY3, EXTI_TRIGGER_BOTH);
exti_init(KEY4, EXTI_TRIGGER_BOTH);

// 设置中断优先级
interrupt_set_priority(KEY_EXTI, 0);

// 获取中断标志
exti_flag_get(KEY1);

// 清除中断标志
exti_flag_clear(KEY1);
```

#### 中断服务函数模板
```c
void key1_exti_handler(void)
{
    if (exti_flag_get(KEY1)) {
        exti_flag_clear(KEY1);
        exti_state[0] = 1;   // 置标志位，在主循环处理
    }
}
```

#### 中断设计模式
```
中断处理原则：
1. 快进快出：中断中只做标记，不做复杂处理
2. 标志位机制：在中断中设置标志，主循环中处理实际逻辑
3. 去抖思路：
   - 硬件去抖：加RC滤波电路
   - 软件去抖：判断两次中断间隔时间，或使用双边沿+延时确认
```

#### 现象说明
- KEY1 上升沿触发：按键释放时LED闪烁1次
- KEY2 下降沿触发：按键按下时LED闪烁2次
- KEY3/KEY4 双边沿触发：不同动作闪烁次数不同

## 第四阶段：周期任务与时间基准

### E05_pit_demo：周期中断定时器

**核心内容**：学习PIT(Periodic Interrupt Timer)实现精确周期任务。

#### 关键API

```c
// PIT初始化（毫秒级）
// 参数：PIT通道, 周期(毫秒)
pit_ms_init(PIT_CH0, 1000);   // 1秒触发一次

// 设置中断优先级
interrupt_set_priority(PIT_IRQn, 0);

// PIT中断服务函数
void pit_handler(void)
{
    pit_state = 1;   // 设置标志位
}
```

#### 主循环处理模式
```c
int main(void)
{
    pit_ms_init(PIT_CH0, 1000);

    while(1) {
        if (pit_state) {
            // 处理定时任务
            gpio_toggle_level(LED1);
            pit_state = 0;
        }
    }
}

void pit_handler(void)
{
    pit_state = 1;
}
```

## 第五阶段：控制输入输出

### E04_pwm_demo：脉宽调制

**核心内容**：学习PWM输出，控制电机、舵机、LED亮度。

#### 关键API

```c
// PWM初始化
// 参数：PWM通道, 频率(Hz), 初始占空比(0-10000对应0-100%)
pwm_init(PWM_CH1, 17000, 0);   // 17KHz, 0%占空比

// 设置占空比
// 参数：PWM通道, 占空比(0-10000)
pwm_set_duty(PWM_CH1, 5000);   // 50%占空比
```

#### 占空比计算
```
占空比 = duty / PWM_DUTY_MAX * 100%
PWM_DUTY_MAX = 10000
duty = 5000 → 50%
```

#### 典型应用
- 电机调速：17KHz频率
- 舵机控制：50Hz频率 (20ms周期)
- LED调光：>1KHz避免闪烁

### E03_adc_demo：模数转换

**核心内容**：学习ADC采样，了解分辨率、均值滤波。

#### 关键API

```c
// ADC初始化
// 参数：ADC通道, 分辨率
adc_init(ADC1_CH3_B14, ADC_12BIT);  // 12位分辨率 (0-4095)
adc_init(ADC1_CH4_B15, ADC_10BIT);  // 10位分辨率 (0-1023)
adc_init(ADC1_CH10_B21, ADC_8BIT);   // 8位分辨率  (0-255)

// 读取ADC值
adc_convert(ADC_CHANNEL1);                    // 单次转换
adc_mean_filter_convert(ADC_CHANNEL, 10);      // 10次均值滤波
```

#### 电压换算
```
12位ADC: 电压 = ADC值 / 4095 * 3.3V
10位ADC: 电压 = ADC值 / 1023 * 3.3V
8位ADC:  电压 = ADC值 / 255  * 3.3V
```

**核心内容**：学习正交编码器和方向编码器模式，用于测速。


## 第七阶段：参数保存

### E08_flash_demo：内部Flash存储

**核心内容**：学习使用内部Flash保存参数，掉电不丢失。

#### 关键概念
- Flash按页擦除（2KB/页）
- 写入前必须先擦除
- 使用联合体(union)灵活存储不同类型

#### 关键API

```c
// Flash初始化
flash_init();

// 检查是否已写入
flash_check(section, page);

// 擦除页
flash_erase_page(section, page);

// 写入数据缓冲区到Flash
flash_write_page_from_buffer(section, page);

// 读取Flash到数据缓冲区
flash_read_page_to_buffer(section, page);

// 清空数据缓冲区
flash_buffer_clear();

// 数据缓冲区（联合体类型）
flash_union_buffer[索引].float_type   = 3.14f;
flash_union_buffer[索引].uint32_type  = 12345;
flash_union_buffer[索引].int16_type    = -32768;
```

#### 使用流程
```c
// 1. 读取已有数据
flash_read_page_to_buffer(SECTION, PAGE);

// 2. 修改数据
flash_union_buffer[0].float_type = 3.14159;

// 3. 写入Flash（自动先擦除）
flash_write_page_from_buffer(SECTION, PAGE);
```

#### 注意事项
- 每个索引只能存储一种类型
- 写入前确认数据对齐
- 断电丢失问题：只有重新上电后读取才是正确数据

## 进阶阶段

### E13_cache_demo：Cache缓存管理

**核心内容**：学习RT1064的L1 Cache机制及数据一致性。

#### 背景知识
- RT1064有32KB L1 Cache
- DMA操作时存在缓存一致性问题
- CPU写入数据 → 先到Cache，未立即到RAM
- DMA从RAM读取 → 可能读到旧数据

#### 关键API

```c
// 禁用/启用DCache
L1CACHE_DisableDCache();
L1CACHE_EnableDCache();

// 清除并失效整个Cache
L1CACHE_CleanInvalidateDCache();

// 按范围操作Cache
L1CACHE_CleanInvalidateDCacheByRange(addr, size);
```

#### 使用场景
- DMA与CPU共享数据时
- 外设通过DMA传输数据时
- 需要确保数据一致性的地方

### E14_code_running_in_itcm_demo：代码运行于ITCM

**核心内容**：学习将关键代码放到ITCM高速运行。

#### 内存分布
| 区域 | 大小 | 特性 |
|------|------|------|
| ITCM | 64KB | 指令紧耦合存储，直接运行代码，无Cache延迟 |
| DTCM | 448KB | 数据紧耦合存储，高速访问 |
| OCRAM | 512KB | 普通内存 |

#### 关键宏

```c
// 将函数放到ITCM
AT_ITCM_SECTION_INIT(void test_function(void))
{
    // 函数代码
}

// 将变量放到DTCM
AT_DTCM_SECTION(uint8 buffer[100]);

// 将变量放到OCRAM
AT_OCRAM_SECTION(uint8 buffer[100]);

// 对齐指定
AT_DTCM_SECTION_ALIGN(uint8 buffer, 64);
```
